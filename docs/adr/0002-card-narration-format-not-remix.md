# 产品形态为推文卡片解说视频，而非素材混剪

视频画面主体是 Tweet Card（仿推文双语卡片）+ TTS 解说 + 字幕，而不是参考资料（参考.md）里那套"素材库视觉匹配混剪"。卡片解说形态砍掉了素材库、镜头匹配、剪映草稿等最重的环节，且与"外网热帖搬运"的内容定位天然一致。

## Consequences

- 不需要维护素材库（assets/），管线里没有视觉匹配环节。
- 参考.md 的混剪工作流仅作为后期增强的参考，不是本项目的骨架。
- Tweet Card 用 HTML 模板渲染（数据来自 API），不做真实页面截图——仿卡片比真截图分辨率可控、无反爬问题，还能嵌入中文翻译。
